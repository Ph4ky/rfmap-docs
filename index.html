Features

📡

Live DOA Tracking

Real-time direction-of-arrival lines from multiple KrakenSDR stations. Auto-refresh every 4 s. DOA lines automatically extend 30% past the farthest intersection point.

🎯

Triangulation

Spherical bearing intersection of two or more station rays. With N stations, N×(N−1)/2 intersection pairs are computed. Confidence-weighted centroid as best estimate.

🗺️

Sovereign Mapping

SwissImage (swisstopo), Ortho (Esri + labels), Swisstopo topographic, Dark and Light basemaps — all tile-cached locally for disconnected operations.

⬡

Simulation Mode

Frontend-only mission planning tool. Simulate TX power, noise, environment loss and GDOP geometry before field deployment. Animation mode visualises measurement scatter in real time. Route simulation moves the target along a multi-waypoint path at a defined speed.

📐

GDOP Optimiser

Computes geometry quality (0–100%) and suggests optimal station positions. Lower GDOP = better — a score near 100% means ideal angular spread. When TX power is known, range constraint based on FSPL limits the deployment radius physically.

📏

Station Distances

Optional layer draws labelled lines between all station pairs — distances in metres or kilometres. Useful for tactical spacing and frequency planning.

🔧

Frequency Advisor

Integrated in the station config modal. Detects known interferers (CH Pager 147.3–147.7 MHz, TETRA, BOS, FM) in the sampling band and recommends the optimal sample rate with one click.

🧭

Azimuth Calibration

Place a calibration marker on a known reference transmitter. The system calculates per-station azimuth corrections applied server-side retroactively to all historical data.

📍

Position Override

Drag station markers to correct GPS errors. Override coordinates are saved and used for DOA lines, triangulation and calibration. New stations auto-placed in the current map area.

🔥

Heatmap

Density map of intersection points over time — shows signal source dwell areas even when individual intersections are noisy.

⚙️

Remote Config & Deploy

Push receiver settings (frequency, gain, DOA method, array type) to all stations simultaneously. SSH-deploy the config sync script directly from the portal.

📱

Mobile Ready

Responsive toolbar, off-canvas panel, full-screen config modal. Dedicated ⬡ button activates simulation on mobile where logo triple-tap isn't accessible.

🔐

User Management

Session-based authentication with three roles: admin (full access), operator (all station controls), viewer (read-only). Admin UI at /admin — create, delete and manage users. Default credentials set on first start.

⏻

Station Enable / Disable

Toggle individual stations on/off without deleting them. Disabled stations are excluded from live DOA lines, triangulation and simulation. State persists in the database.

🔘

Hardware Shutdown Button

GPIO 17 (Pin 11) shutdown button support for KrakenSDR Raspberry Pi stations. Deployed automatically alongside the config sync script. Hold 2 seconds → clean shutdown. Compatible with IMU on I2C (GPIO 3 / SDA intentionally avoided).

Physics Model

FSPL = 32.44 + 20·log₁₀(d_km) + 20·log₁₀(f_MHz) [dB, free space] P_rx = TX_dBm − FSPL − env_loss_dB RX threshold: −110 dBm (detection) / −100 dBm (DOA-quality, SNR ≥ 10 dB) Example — 100 mW (+20 dBm) TX at 148.537 MHz: Free field (+0 dB loss) → ~40 km range Suburban (+20 dB loss) → ~4 km range Urban/Forest (+30 dB loss) → ~1.3 km range

Route Simulation — Moving Target

Plan transmitter movement before a field deployment — especially useful with only 2 stations to find paths with acceptable GDOP throughout.

Workflow: 1. Place target marker at start position → click +Pkt 2. Move marker to next point (bend, road junction) → click +Pkt 3. Repeat for every waypoint — any number supported 4. Set speed (km/h) and enable Animate → marker travels along the route, predict polygon accumulates On the map: Green circle = waypoint 1 (start) Yellow circles = intermediate waypoints (numbered) Red circle = last waypoint (end) White dashed line + ∑ distance label connects all points ↩ removes last waypoint · ✕ clears entire route

GDOP Geometry

GDOP (Geometric Dilution of Precision): lower value = better geometry. Score 100% = ideal angular spread between stations.

Ideal cut angle = 180° / N stations N=2 → 90° N=3 → 120° N=4 → 45° (semicircle) Position error ≈ distance × tan(noise_angle) 500 m @ 5° noise → ~44 m error 2 km @ 5° noise → ~175 m error 5 km @ 10° noise → ~882 m error

System Architecture

Hornet Sentinel follows a Distributed Edge Computing philosophy — signal processing happens on the sensor, only compact bearing vectors reach the portal. The backend runs on your own server (on-premise) or as a hosted service — same codebase, your choice.

graph TD subgraph FIELD["Field Site (Edge)"] A[Antenna Array 5 elements] -->|IQ Data| B(KrakenSDR + Raspberry Pi) B -->|FFT / MUSIC algorithm| C{Bearing Vector JSON} end subgraph TRANSPORT["Secure Transport"] C -->|HTTPS + API Key| D[ZeroTier VPN / Internet] end subgraph BACKEND["Backend — Cloud or On-Prem"] D --> E[Linux Server · Docker] E --> F[(PostgreSQL)] E --> G[Web Portal] end style B fill:#f96,stroke:#333,stroke-width:2px style G fill:#2ecc71,stroke:#333,stroke-width:2px style D fill:#3498db,stroke:#333,stroke-width:1px style E fill:#2ecc71,stroke:#333,stroke-width:1px

The KrakenSDR Pi processes raw IQ data locally — only the resulting bearing vector (a few bytes of JSON) is transmitted. This allows operation over LTE with minimal bandwidth.

The portal is currently available as a hosted service — self-hosting via Docker Compose is supported with the same feature set.

Why Edge Processing?

Raw IQ stream → ~50–200 MB/s (unusable over LTE) Bearing vector → ~0.5 KB/report (trivial over LTE / satellite) KrakenSDR Pi processes locally: 1. FFT — frequency domain conversion 2. MUSIC algorithm — direction-of-arrival estimation 3. GPS timestamp + heading fusion → POST /save.php {station_id, lat, lon, doa_angle, confidence, ...}

Architecture

KrakenSDR Pi (mobile) ├── GPS + Compass → absolute bearing ├── POST /save.php → Portal (HTTP, VPN tunnel) └── krakensdrd_config_sync.py (polls for config changes) Portal (Docker) ├── Node.js (Express) — REST API ├── PostgreSQL + PostGIS — data storage ├── Nginx — HTTPS :8443 / HTTP :8080 └── Static frontend — Leaflet.js, dark theme

The system is VPN-agnostic — ZeroTier is used in the reference setup but any overlay network (WireGuard, OpenVPN, IPsec) works equally well.

FAQ

Why does the error polygon change size when I move the target?

Two geometric effects: cut angle (GDOP) — lines near-parallel to each other produce a huge, unstable intersection. And distance — same angular noise at 5 km = 5× larger position error than at 1 km. Optimal: stations surrounding the target with ~60–90° cut angles.

Why does a station disappear in simulation?

The FSPL model computes received power. If it drops below −110 dBm (KrakenSDR sensitivity) the station «hears nothing». Increase TX power, reduce environment loss, or move station closer to target.

What is TX Power = 0?

Setting TX to 0 means «unknown». The GDOP optimizer runs without a range constraint (no orange range circle). The simulation itself uses 1000 mW internally so stations remain visible.

What does the Frequency Advisor do?

It checks whether known interferers (CH Pager 147.3–147.7 MHz, TETRA, BOS, FM) fall into the sampling band. It then recommends the smallest KrakenSDR-compatible sample rate that excludes all detected interferers. For CH 148.537 MHz: sample rate ≤ 1.024 MHz keeps CH pagers out of band.

Does the system work offline?

Yes. All map tiles are cached locally (/app/tile-cache/). The system is designed for disconnected operations.

How many stations are recommended?

Minimum 2 (1 intersection). Optimal 3 (3 intersections, error triangle visible). Ideal 4+ (6+ intersections, robust even if one station fails).

How does azimuth calibration work?

Place a test transmitter at a known position, drag the calibration marker to that position, and click «Korrekturen übernehmen». The correction is applied server-side at query time — retroactively fixes all historical data without reprocessing.

How does route simulation help with 2-station geometry?

With only 2 stations you get a single bearing intersection. Cut angle (and therefore position error) varies drastically depending on where the target is relative to the baseline. Use route simulation to walk candidate transmitter paths and watch the GDOP quality badge live — green means the geometry holds, red means the target is in a blind zone. Choose the field path that keeps the badge green longest.
